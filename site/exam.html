Assessment

Exam

Respond to an RFP by building a visual artifact that supports a clear angle, and write the strategy section of a proposal that connects your data work to the funder's problem.

What you will submit

  1. Strategy section of a proposal
    Following the example provided. This covers: context, objectives, approach, proposed activities, and deliverables.
  2. Visual artifact
    A map, dashboard, or chart published on GitHub Pages that supports a specific argument. Link to the visuals that support your case — not a full project landing page.
  3. Data
    The data behind the artifact, visible in your GitHub repository.

What is NOT expected

  • Logistical section — no timeline, team composition, or activity sequencing
  • Financial proposal — no budget
  • Step-by-step process documentation — but your data must be in the repository and data quality will be inspected

AI use

You are expected to use AI tools. The exam tests logic and methodology, not memory. The question is whether you can demonstrate human-led reasoning:

  • Did you choose meaningful data sources?
  • Did you verify and clean the data?
  • Does your visual artifact support your angle?
  • Is there a clear and consistent storytelling in your proposal?

How to submit

Submit a PDF that includes links to your GitHub Pages site and your GitHub repository.
